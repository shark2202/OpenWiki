内容 + 数据 合并方案

三个方案对比，砍掉"数据"标签页，功能全部融入"内容"

方案 A 日期筛选条 — 最少改动

在内容列表顶部加一行日期筛选器（类似"今天 / 本周 / 本月 / 选日期"），导出按钮放在右侧。
类型筛选 + 日期筛选共存，用户可以同时过滤"本周的链接"。最轻量的方案。

改动最小，几乎只加一行 UI 用户习惯不变，内容列表逻辑保持 日期+类型双维度筛选
没有日期导航的全局视野 想看很久以前的内容需要手动选日期

方案 B 时间线 — 日期做分隔线

取消独立的日期侧边栏，改成在内容流中插入日期分隔线。每个日期段可折叠，右侧有"导出此日"小按钮。
像聊天记录一样自然的时间流，一目了然看到"今天 8 条、昨天 5 条"。

时间感最强，像日记一样浏览 单列布局，移动端友好 日期+数量一目了然
内容多时滚动距离长 跳到某一天需要大量滚动

方案 C 可收起日期侧边栏 — 两种模式

内容列表左侧加一个可收起的日期侧边栏（默认收起）。点击"日历"图标展开，选择日期后内容列表过滤到那一天。
相当于把数据板块的日期导航嵌入内容板块，想用就展开，不用就收起。导出功能放在侧边栏底部。

保留了日期导航的全局视野 默认收起不占空间 两种使用模式，灵活度最高
实现复杂度最高 展开后内容区域变窄
