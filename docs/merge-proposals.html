小云 数据 + 内容融合方案

把数据板块的功能融入内容板块，减少一个 Tab，同时保留日期浏览和导出能力。

数据板块有什么是内容板块没有的？

数据板块独有功能

  • 按日期浏览（选某一天看内容）
  • 导出为 Markdown 文件
  • 统计信息（总条数、天数）
  • 按类型分组（某一天内的文本/链接/图片）

融合后要保留的

  • 按日期筛选内容
  • 导出功能入口
  • 统计可以弱化或移到设置
  • 类型分组：内容板块已有类型筛选
方案 A 日期横条

在筛选栏旁边加一个日期按钮，点击展开一排日期横条。选了日期就只看那天的内容，再点一下取消筛选回到全部。导出按钮放在右上角。最小改动。

方案 B 窄侧栏

左边加一个 140px 的窄日历侧栏，保留数据板块"按月展开、按日选择"的交互，但更紧凑。默认看全部，选了日期就切到那天。侧栏始终可见，像 Finder 的边栏。

方案 C 日期分组

不加侧栏也不加横条，直接在内容列表里按日期分组，每天一个小标题。滚动浏览时自然看到日期分界。导出按钮放在筛选栏右侧。最简单，零新增 UI 元素。
