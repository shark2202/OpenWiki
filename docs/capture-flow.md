# 小云内容捕获流程图

```
用户复制内容
     │
     ▼
┌─────────────────────┐
│ ① 剪贴板检测         │  clipboard.rs
│ 每 500ms 轮询        │  检测文本/图片变化
│ 计算 hash 去重       │  识别来源 App
└─────────┬───────────┘
          │ emit: capture:clipboard
          ▼
┌─────────────────────┐
│ ② 事件去重           │  detector.rs
│ 3秒窗口内相同内容     │  如果重复 → 丢弃
│ 跨来源去重(截图+剪贴板)│
└─────────┬───────────┘
          │ 新内容通过
          ▼
┌─────────────────────┐
│ ③ 模式判断           │  detector.rs
│                     │
│ ┌─确认模式(默认)──┐  │
│ │ 存入 pending    │  │
│ │ 显示气泡窗口    │  │
│ │ 等待用户确认    │  │
│ └────────────────┘  │
│                     │
│ ┌─自动模式────────┐  │
│ │ 直接保存        │  │
│ └────────────────┘  │
└─────────┬───────────┘
          │ 用户点击确认 / 自动保存
          ▼
┌─────────────────────┐
│ ④ 保存到数据库       │  capture.rs: save_content_auto()
│ 生成 UUID           │
│ 检测内容类型(文本/链接/图片) │
│ 检测 URL            │
│ 计算 hash 防重复     │
│ 写入 SQLite         │
└─────────┬───────────┘
          │
          ├──────────────────────────────────┐
          │                                  │
     ┌────┴────┐  ┌──────────┐  ┌───────────┴──────────┐
     │ 文本内容 │  │ 图片内容  │  │ 链接内容              │
     └────┬────┘  └────┬─────┘  └───────────┬──────────┘
          │            │                     │
          │       ┌────┴─────────────┐  ┌────┴──────────────────┐
          │       │ ⑤a OCR 文字识别   │  │ ⑤b 网页内容读取        │
          │       │ macOS Vision     │  │ url_reader.rs          │
          │       │                  │  │                        │
          │       │ emit:            │  │ 微信? → 直接HTML/       │
          │       │ content:ocr-done │  │        content_noencode│
          │       └────┬─────────────┘  │ Twitter? → fxtwitter   │
          │            │                │ 其他? → Jina Reader     │
          │            │                │                        │
          │            │                │ emit:                  │
          │            │                │ content:url-fetched    │
          │            │                └────┬───────────────────┘
          │            │                     │
          ▼            ▼                     ▼
┌─────────────────────────────────────────────────────┐
│ ⑥ AI 标签 + 摘要生成                                 │
│ capture.rs: spawn_summary_task()                     │
│                                                     │
│ 条件: 内容 ≥ 6 字节 且 API Key 已配置                 │
│ 取前 500 字 → 调用 AI API → 返回 JSON                │
│ {"tags":["标签1","标签2"], "summary":"一句话摘要"}     │
│                                                     │
│ emit: content-summary-ready                          │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│ ⑦ 前端刷新显示                                       │
│ ContentList.tsx 监听三个事件:                          │
│   • content:url-fetched    → 重新加载列表             │
│   • content:ocr-done       → 重新加载列表             │
│   • content-summary-ready  → 重新加载列表             │
│                                                     │
│ ContentCard.tsx 显示状态:                              │
│   • 链接: 读取中 → 正文+标签+摘要 / 读取失败+重试      │
│   • 图片: 缩略图 → OCR文字+标签+摘要                  │
│   • 文本: 原文+标签+摘要                              │
│   • AI分析中: 有内容但标签未出(仅最近2分钟的内容)       │
└─────────────────────────────────────────────────────┘
```

## 各环节耗时参考

| 环节 | 耗时 | 可能卡住的原因 |
|------|------|---------------|
| ① 剪贴板检测 | <500ms | - |
| ② 事件去重 | <1ms | - |
| ③ 气泡显示 | ~150ms | - |
| ④ 保存数据库 | <50ms | 重复内容被拦截 |
| ⑤a OCR | 1-3秒 | 大图片 |
| ⑤b 网页读取 | 2-15秒 | 微信新格式、Jina超时、网络慢 |
| ⑥ AI分析 | 3-10秒 | API限流(429)、余额不足(402)、Key错误 |
| ⑦ 前端刷新 | <100ms | 事件未监听 |
