1
剪贴板监听
检测你复制的内容
< 0.5 秒
它在做什么
小云每隔 0.5 秒检查一次你的剪贴板，看你有没有复制新内容。就像一个安静的助理一直在旁边等着。
它能检测到
  • 文字内容（文本、链接）
  • 图片内容（截图、复制的图片）
  • 来自哪个 App（微信、Chrome、Safari 等）
你感受到的
完全无感。你正常复制东西，小云在后台默默检测。
2
去重过滤
避免重复保存相同内容
瞬间
它在做什么
检查这个内容是不是 3 秒内刚保存过。比如你截图的时候，剪贴板和截图目录可能同时触发，小云只保留一份。
去重方式
  • 文字：对比前 64 个字符
  • 图片：对比尺寸（宽 x 高）+ 文件路径
  • 已保存过的内容（数据库 hash 比对）
你感受到的
无感。重复内容被静默丢弃，不会弹出气泡。
3
确认 / 自动
根据你的设置决定怎么处理
~0.15 秒
两种模式
  • 确认模式（默认）：弹出一个小气泡，等你点击确认才保存。倒计时结束后自动消失（默认丢弃）
  • 自动模式：所有复制的内容直接保存，不需要你操作
气泡窗口
气泡出现在屏幕角落，不会抢走你正在用的 App 的焦点。你可以在设置里选择气泡的样式（圆形/长条）和位置（六个方向）。
你感受到的
确认模式下：屏幕右下角出现一个小气泡，显示你刚复制的内容预览。点击气泡 = 保存，不管它 = 丢弃。
4
保存到本地数据库
内容安全地存在你的电脑上
< 50 毫秒
它在做什么
  • 生成一个唯一 ID
  • 自动判断内容类型：纯文本、链接、还是图片
  • 如果是链接，提取出干净的 URL
  • 如果是图片，复制到专用文件夹 + 生成缩略图
  • 记录来源 App、时间、大小
  • 再次检查是否重复（数据库级别）
  • 写入本地 SQLite 数据库
你感受到的
内容出现在小云主窗口的「内容」列表里。此时链接类型会显示"读取中..."，因为正文还没抓取。
第 5 步：内容处理（根据类型分三条路）
📝
纯文本
复制的文字段落、笔记等
无需处理
处理方式
文字内容不需要额外处理，直接进入下一步（AI 分析）。保存时已经存好了原文。
🖼️
图片
截图、复制的图片
1-3 秒
处理方式
用 macOS 系统的 Vision 框架做 OCR（光学字符识别），把图片里的文字提取出来。
结果
  • 成功：图片下方会显示识别出的文字
  • 失败：图片里没有文字（纯照片），静默跳过
🔗
链接
微信文章、网页、推文等
2-15 秒
处理方式
根据链接来源选择最优的读取方式：
  • 微信文章：直接抓 HTML，尝试两种提取方式（传统格式 + 新格式）
  • Twitter/X：通过 fxtwitter API 获取
  • 其他网页：通过 Jina Reader 阅读
常见问题
微信文章有时读不出正文（新版动态渲染格式）。Jina Reader 可能被微信人机验证挡住。此时只能拿到标题。
6
AI 生成标签 + 摘要
用 AI 分析内容，生成标签和一句话摘要
3-10 秒
触发条件
  • 内容长度 ≥ 6 字节（约 2 个中文字）
  • AI API Key 已配置
它在做什么
把完整内容发给你配置的 AI 模型（阿里云百炼/OpenRouter/Anthropic/OpenAI），让它通读全文后返回：
  • 2-4 个标签：主题词 + 内容类型 + 价值点
  • 一句话摘要：不超过 50 字的中文摘要
你感受到的
内容卡片上先显示"AI 分析中..."（闪烁的标签），几秒后变成真正的标签和摘要。
常见问题
没有 API Key：静默跳过，不生成标签。
429 限流：免费模型排队太多，换个模型试试。
402 余额不足：OpenRouter 余额用完了。
模型不兼容：部分模型不支持 JSON 返回格式。
7
界面显示更新
内容列表自动刷新，展示最终结果
< 0.1 秒
刷新时机
小云监听三个信号，收到任何一个就自动刷新内容列表：
  • 网页读取完成 → 显示正文预览或错误信息
  • OCR 识别完成 → 图片下方显示识别文字
  • AI 标签摘要完成 → 显示标签 + 一句话摘要
最终显示状态
  • 链接：标题 + 标签 + 摘要 + "展开原文"按钮
  • 图片：缩略图 + OCR 文字 + 标签
  • 文本：原文预览 + 标签 + 摘要
完成 — 内容安全保存在你的电脑上
